# Установка CNCTD OS в n8n

## 1. Подключить Notion

1. В Notion откройте каждую из 9 баз отделов и базу **🧭 Диспетчер недели**.
2. Нажмите `•••` → `Connections` и добавьте ту же Notion integration, токен которой будет использовать n8n.
3. В n8n откройте `Credentials` → `New credential` → `Header Auth`.
4. Название: `CNCTD Notion API`; Header Name: `Authorization`; Header Value: `Bearer secret_...`.

Токен не вставляйте в workflow и не коммитьте в GitHub.

## 2. Импортировать workflow

1. Откройте пустой workflow: https://automation.cnctd.fi/workflow/11Hro-yuBqQ9d0ISHeZzf
2. Меню `•••` (справа сверху) → `Import from File`.
3. Выберите `workflows/01-weekly-dispatcher.json` из этого репозитория.
4. В каждом узле `Читать: ...`, а также `Найти строку дня` и `Обновить диспетчер`, выберите credential `CNCTD Notion API`.
5. Нажмите `Test workflow`. Проверьте, что в Notion появились/обновились ровно 7 строк — ПН–ВС.
6. Если тест успешен, включите workflow кнопкой `Active`.

## Что делает v1

- ежедневно в 06:00 по Europe/Kiev читает девять баз отделов;
- не копирует задачи в недельный план;
- считает открытые, срочные и просроченные задачи;
- распределяет направления по дням с балансировкой нагрузки;
- отправляет на выходные только направления с дедлайном до понедельника;
- создаёт или обновляет строку каждого дня (idempotent upsert);
- строит шкалу `████░░░░░░`.

## Ограничения v1

Notion API возвращает до 100 активных задач на отдел за один проход. Перед превышением лимита нужно добавить pagination sub-workflow. Аналитика, уведомления и AI-классификация должны быть отдельными workflow и не входят в первый безопасный релиз.

## Журнал и аналитика

Импортируйте в **два отдельных пустых workflow** файлы `02-dispatcher-event-journal.json` и `03-weekly-analytics.json`. Назначьте `CNCTD Notion API` каждому HTTP-узлу. Журнал запускается ежедневно в 06:15 после диспетчера; аналитика — по воскресеньям в 20:00. Сначала вручную выполните workflow журнала, затем аналитику.

## План сотрудников

Импортируйте `04-team-daily-plan.json` в отдельный пустой workflow и назначьте credential всем HTTP-узлам. Он запускается в 06:10, читает назначения диспетчера и группирует активные задачи по категории и текстовому полю `Ответственный`. В `Задачи ПН–ПТ` и `Задачи выходные` публикуется одна строка на комбинацию день + категория + исполнитель. Для Ильи также заполняется поле Person; остальные имена отображаются в поле `Исполнитель`, пока пользователь не приглашён в Notion workspace.
